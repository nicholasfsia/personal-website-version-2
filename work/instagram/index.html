Instagram WebApp Redesign

Role: User Interface and User Experience Designer

Tools: Photoshop and Illustrator

Methods: Human Factors in Design, Design Sprints

Living in this era, humans are surrounded by technologies that are meant to help human's needs and desires. However, sometimes those technologies were made for the sake of technology, not human. In this project, Instagram WebApp was chosen as a sample case. It was improved using human factors methodologies and design sprints, to develop a better and more intuitive web application.

Goal

This Project is part of the ‘Human Factors in Design’ course. The goal is to choose a product, either software or hardware that has a computer display and redesign it using human factors methodology.

Approach

Instagram WebApp was chosen as the sample product. The process consisted of 7 design sprints including Situation of Concern (SOC), persona development, design walkthroughs, heuristic evaluations, Hierarchical Task Analysis (HTA), Fault Tree Analysis (FTA), and display/control compliance.

These are some of the problems and recommendations of the Instagram WebApp:

Safety

User accounts are public by default and privacy setting is not available on the Instagram web application. This violates Nielsen’s UI design principle of user control and freedom. Add a privacy status setting in the edit profile under the Privacy Information will overcome this issue (R1).

Performance

Navigation bar does not appear as the user scrolls through their feed which requires them to scroll all the way back to the top to access the navigation bar. This is an example of minimizing information on access cost, which is a display principle. The cost, in this case, is user’s effort to scroll up to the top and that required extra step. In terms of usability, this is a bad practice. It requires the user to work more, yet in fact it, can be resolved by simply showing it at all times when the user is scrolling, making it easier for them to access their profile, notification or search menu. Other option, show the navigation bar when the user scrolls up, and make it disappear when the user scrolls down to increase the content view (R6).

New notification is not indicated in any form. This is an example of feedback issue, that Norman discussed in his book the “The Design of Everyday Things”. Changing the website heading icon interface (from empty heart to red/pink heart button) and or adding sound effects to alert the user of notifications will fix that issue (R9).

When changing profile picture, photo there is a lag in responsiveness once the picture is uploaded. The user must refresh their profile to see any changes that they have made. There is no direct visual feedback, thus leads to no visibility on system status (Nielsen). The profile picture should be updated immediately, or at least have a pop-up “Saved” message so the user does not have to guess if the picture has been uploaded or not (R8).

There is also no visual component that signifies the user towards where to change their profile picture. According to Norman’s principle, this is considered as a signifiers problem. By adding an overlapping icon on top of the profile picture to indicate the feature of changing profile, the display signifies the users where to navigate to change their profile picture (R11).

When attempting a logout, the user is required to go to user profile instead of logging out from any pages. In terms of control, this increases decision complexity. This adds extra steps for the user to accomplish a task and also there’s a high chance the user doesn’t know where to go for logging out. Profile thumbnail should pop up, or enlarge when it’s clicked to give the user log out option from any pages (R13).

Preference

Just like the mobile app, user prefers to be recommended while exploring pictures on Instagram. However, the website app doesn’t have this feature. As mentioned in Norman’s design principle, consistency is an important part of a design. This is an example of inconsistency. To make a more consistent and better user experience, explore feature can be made available (explore feature is picture recommendation based on user’s likes, comments, and history) (R18).

What's next?

Re-evaluate some of the unnecessary redesigns and reconsider some of the redesign trade-offs. Propose some of the redesign recommendations to Instagram.
